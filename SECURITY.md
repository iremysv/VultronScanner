# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.x.x (dev) | ✅ Active development |

---

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub Issues.**

If you discover a security vulnerability in VultronScanner, please report it responsibly:

1. Open a [GitHub Security Advisory](https://github.com/iremysv/VultronScanner/security/advisories/new) (preferred)
2. Describe the vulnerability in detail, including:
   - A clear description of the issue
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

### What to Expect

- **Acknowledgement** within 48 hours
- **Status update** within 7 days
- **Fix timeline** communicated as soon as assessed

---

## Responsible Disclosure

We follow the principle of responsible disclosure. We ask that you:

- Allow reasonable time for us to fix the issue before public disclosure
- Avoid exploiting the vulnerability beyond what is necessary for demonstration
- Do not access, modify, or delete data belonging to others

---

## Scope

This security policy covers the VultronScanner codebase itself.  
It does **not** cover vulnerabilities in:
- Third-party dependencies (report those upstream)
- Systems scanned using VultronScanner (that is the user's responsibility)

---

## Legal Notice

VultronScanner is designed exclusively for use on systems you own or have explicit written permission to test. Unauthorized use against systems you do not own is illegal. The developers assume no liability for misuse.
